Test Case TC_110

This page describes one of the test cases of the EODiSP. The test case described in this page instantiates a nominal scenario of the use case UC_110.

This test case uses the EarthCARE Simulator as a test platform. The simulator is operated in its default configuration.

Contents

  • Overview
  • Steps

Overview

Identifier TC_110
Name Abort a running EarthCARE Simulation Experiment
Primary Actor PnP_1 Simulation Owner
Description Describes the steps to abort a running instance of the EarthCARE Simulator.
Pre-Condition Test case TC_106 (Run EarthCARE Simulation Experiment) has partially completed. This means that the experiment has been started and is in running state but has not yet completed executing.
Post-Condition The EarthCARE simulation experiment is in a clean stopped state and is ready to be started again.

top ^

Steps

  1. Actor right-clicks on the experiment node and selects Interrupt/Cancel Experiment.
  2. System immediately stops the execution of the experiment by first issue a federate stop request to all involved model manager applications and by stopping all locally executing tasks for the experiment.
  3. System shows the state of of the experiment when terminating.
  4. After stopping all tasks, the system enters a clean state in which it is ready to execute the experiment again.
