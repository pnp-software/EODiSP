EarthCARE Simulator Configuration

Some of the test cases defined for the EODiSP use the EarthCARE Simulator as a demonstration platform. This page describes the configuration of the EarthCARE Simulator that is assumed for the test cases.

Model Repositories

Two simulation repositories have been created for the EODiSP test cases: a local repository useful for operating the EODiSP in local mode, and a remote repository useful for operating the EODiSP in remote mode.

The local repository is installed on the platform of the ETH_2 federate owner.

The remote repository is installed on the platform of the ETH_3 federate owner.

Simulator Federates

Two simulation federates are installed on the platforms of the federate owner. The simulation requires all federates to be installed at the same relative path on their respective platform. This path is TBD.

All simulation federates are configured to be unconditionally available to all simulation managers with the exception of the TBD federate that is configured not to be available to the PnP_1 simulation manager (see next section). This allows the simulator to be used to test the ability of the EODiSP to limit availability of a certain federate to selected simulation managers.

A number of EODiSP test cases require a simulation experiment to be run. The simulation experiment is referred to as 'EarthCARE Simulation Experiment'. The experiment uses all the models of the EarthCARE Simulator. Its FOM can be downloaded from here.
